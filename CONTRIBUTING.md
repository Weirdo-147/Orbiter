# Contributing to Orbiter

Thanks for your interest in contributing.

## Workflow
1. Open an issue or discuss design before large changes.
2. Create a focused branch.
3. Keep PRs small and scoped.
4. Include docs updates for user-visible behavior.

## Commit guidance
- Use concise, imperative commit messages.
- Prefer prefixes like `feat:`, `fix:`, `docs:`, `chore:`.

## Local commands
- `make bootstrap`
- `make fmt`
- `make lint`
- `make test`

## Code review expectations
- Add rationale and tradeoffs for non-obvious decisions.
- Include follow-up tasks for deferred work.
